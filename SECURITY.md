# Security Policy

## Reporting

If you discover a security issue in this repository, please avoid opening a public issue with exploit details.

Instead, document:

- affected endpoint or policy path
- request pattern that triggers the issue
- expected vs actual behavior
- potential operational impact

Then share the details privately through the appropriate contact channel.

## Scope

This project demonstrates:

- edge request governance
- policy evaluation logic
- rate pressure handling
- redirect and geo routing decisions

No production secrets should be committed to this repository.
